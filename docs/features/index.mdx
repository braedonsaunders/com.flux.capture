Flux Capture includes a comprehensive set of features designed to automate your accounts payable workflow. This overview introduces each major feature and links to detailed guides.

## Core Features

### Smart Extract

Our AI-powered extraction engine reads your documents and automatically captures:

- Vendor name and address
- Invoice number and dates
- Line items with descriptions, quantities, and amounts
- Tax amounts and totals
- PO references and payment terms

[Learn more about Smart Extract →](/docs/capture/features/smart-extract)

### Learning Engine

Flux Capture gets smarter over time by learning from your corrections:

- Remembers vendor-specific date formats
- Learns account coding patterns
- Builds vendor name aliases
- Improves accuracy with each processed document

[Learn more about the Learning Engine →](/docs/capture/features/learning-engine)

### Fraud Shield

Built-in fraud and error detection protects your business:

- Duplicate invoice detection
- Duplicate payment prevention
- Amount anomaly detection
- Benford's Law analysis
- Vendor pattern monitoring

[Learn more about Fraud Shield →](/docs/capture/features/fraud-shield)

### Side-by-Side Review

Review documents efficiently with our split-screen interface:

- Original document on one side
- Extracted data on the other
- Click-to-navigate synchronization
- One-click approval

[Learn more about Side-by-Side Review →](/docs/capture/features/side-by-side-review)

### Email-to-Invoice

Forward invoices to your dedicated email address for automatic processing:

- Unique inbox per installation
- Automatic attachment extraction
- Trusted sender configuration
- Direct-to-queue workflow

[Learn more about Email-to-Invoice →](/docs/capture/features/email-to-invoice)

## How Features Work Together

Flux Capture features are designed to work as an integrated system:

1. **Ingest** - Documents arrive via upload or email
2. **Extract** - Smart Extract reads all data
3. **Validate** - Fraud Shield checks for issues
4. **Review** - Side-by-Side interface for verification
5. **Learn** - Corrections feed the Learning Engine
6. **Create** - Approved documents become transactions

This flow ensures accurate, efficient processing while continuously improving.

## Getting the Most from Features

### Maximize Extraction Accuracy

1. Use high-quality scans (300 DPI recommended)
2. Ensure documents are not skewed or cropped
3. Prefer PDF over image formats
4. Process vendor invoices consistently

### Leverage Learning

1. Always correct errors rather than skipping them
2. Use consistent vendor names
3. Review and approve documents promptly
4. Allow the system time to build patterns

### Optimize Fraud Detection

1. Enable all relevant anomaly checks
2. Review flagged items carefully
3. Adjust thresholds based on your business
4. Monitor the anomaly dashboard
