This guide walks you through installing Flux Capture in your NetSuite environment. The installation process is straightforward and typically takes just a few minutes.

## Before You Begin

Ensure you have:

- Administrator access to your NetSuite account
- SuiteScript enabled
- SuiteCloud CLI configured for your account
- OCR/AI provider credentials for the provider you plan to use

## Installation Steps

### Step 1: Clone the Repository

```bash
git clone https://github.com/braedonsaunders/com.flux.capture.git
cd com.flux.capture
npm install
```

### Step 2: Authenticate SuiteCloud CLI

```bash
suitecloud account:setup
```

### Step 3: Deploy the SDF Project

```bash
cd src
suitecloud project:deploy
```

### Step 4: Verify Installation

Once complete, verify the installation:

1. Navigate to **Customization > Scripting > Scripts**
2. Search for "Flux" and confirm the Suitelet, RESTlet, Map/Reduce, Scheduled Script, User Event, and Email Capture Plugin exist
3. Navigate to **Customization > Lists, Records & Fields > Record Types**
4. Confirm the Flux document and configuration records exist
5. Open the Flux Capture Suitelet

## Accessing Flux Capture

After installation, Flux Capture is accessible in several ways:

### From the Menu
Look for "Flux Capture" in your NetSuite center navigation. Click to open the main application.

### From a Shortcut
Your administrator can add Flux Capture to your shortcuts for quick access.

### Direct URL
Access Flux Capture directly using the Suitelet URL provided after installation.

## Post-Installation Setup

After installing, complete these essential setup steps:

### 1. Configure Extraction Settings

Choose your preferred document extraction provider:
- **OCI Document Understanding** - Native NetSuite integration
- **Azure Form Recognizer** - Microsoft's advanced AI
- **Mindee** - Specialized invoice extraction

See [Configuration](/docs/capture/configuration) for detailed setup instructions.

### 2. Configure Transaction Forms

Upload a NetSuite custom transaction form XML export or use the built-in schema extractor so Flux Capture can render your custom body fields, custom column fields, sublists, and transaction layout.

### 3. Test with a Sample Document

1. Upload a sample invoice to verify extraction is working
2. Review the extracted data
3. Make any needed configuration adjustments

## User Roles and Permissions

Flux Capture respects NetSuite's role-based access control. Ensure users have appropriate permissions:

| Permission | Required For |
|------------|--------------|
| View File Cabinet | Uploading and viewing documents |
| Create Vendor Bills | Approving documents as vendor bills |
| Create Expense Reports | Processing expense documents |
| Custom Records | Accessing Flux Capture records |

> ⚠️ **Warning**: Users without appropriate permissions may see errors when trying to access certain features.

## Updating Flux Capture

When updates are available:

1. Pull the latest repository changes
2. Review the diff and release notes
3. Deploy the updated SDF project with SuiteCloud CLI

> ℹ️ **Info**: Updates typically take 2-5 minutes. Plan updates during low-usage periods.

## Uninstalling

If you need to remove Flux Capture:

1. Remove or undeploy the Flux Capture SuiteCloud project from your NetSuite account
2. Delete the Flux Capture script deployments if needed
3. Delete Flux Capture custom records only after exporting any data you want to keep

> 🚨 **Danger**: Uninstalling removes all Flux Capture data including document records, settings, and learned preferences. This cannot be undone.

## Troubleshooting Installation Issues

### Bundle Not Found

Flux Capture is distributed as source. Deploy it with SuiteCloud CLI from the repository or from a generated SDF archive.

### Installation Failed

If installation fails:
- Check for pending SuiteCloud deployments
- Ensure you have administrator permissions
- Review the SuiteCloud deployment log for the failing object or script

### Missing Menu Items

If Flux Capture doesn't appear in your menu after installation:
- Clear your browser cache
- Log out and back into NetSuite
- Verify your role has access to the Flux Capture SuiteApp
